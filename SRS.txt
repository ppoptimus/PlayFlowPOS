เอกสารแสดงความต้องการระบบ (Software Requirement Specification - SRS)
โครงการ: ระบบบริหารจัดการร้านนวดและสปา (Spa & Massage Management System - Multi-Branch)

1. ภาพรวมระบบ (System Overview)
ระบบ Web Application สำหรับบริหารจัดการร้านนวดและสปาแบบครบวงจร รองรับโครงสร้างแบบหลายสาขา (Multi-Branch) มีระบบจัดการทรัพยากร (หมอนวด, ห้อง), ระบบขายหน้าร้าน (POS), ระบบสมาชิกและแพ็กเกจ, รวมถึงการออกรายงาน

2. สิทธิ์การใช้งาน (Role-Based Access Control - RBAC)
- Super Admin (HQ): เข้าถึงได้ทุกสาขา ตั้งค่าระบบโครงสร้างหลัก ดูรายงานรวมได้ทั้งหมด
- Branch Manager: เข้าถึงได้เฉพาะสาขาของตนเอง จัดการพนักงาน คิว และดูรายงานประจำสาขา
- Cashier / Reception: จัดการจองคิว, POS คิดเงิน, และจัดการสต็อกย่อย
- Masseuse (หมอนวด): ดูตารางคิวงานของตนเอง และยอดรายได้/คอมมิชชัน

3. รายละเอียดฟังก์ชันการทำงาน (Functional Requirements)
โมดูลที่ 1: POS / คิดเงิน (Point of Sale)
- UI สำหรับหน้าจอสัมผัส (Touch-friendly) และ Desktop
- ตะกร้าสินค้า: เพิ่มรายการ "บริการนวด" และ "สินค้าขายปลีก" ลงในบิลเดียวกันได้
- ส่วนลด: รองรับการใส่ส่วนลดท้ายบิล (% หรือ จำนวนเงิน) และส่วนลดราย Item
- การรับชำระ (Payment Gateway): รองรับ เงินสด, โอนเงิน/QR PromptPay, บัตรเครดิต
- ผูกบิลกับ "รหัสหมอนวด" เพื่อนำไปคำนวณคอมมิชชัน


โมดูลที่ 2: Dashboard (กระดานข้อมูลสรุป)
- แสดงข้อมูล Real-time ประจำวันของสาขาที่ Login
- แสดงตัวเลข: ยอดขายรวมวันนี้, จำนวนลูกค้าที่รับบริการวันนี้
- แสดงอันดับ (Leaderboard): หมอนวดที่ทำยอดเงินสูงสุด (Top Earner) 3 อันดับแรก
- แสดงสัดส่วน (Pie/Bar Chart): บริการที่ขายดีที่สุดประจำวัน/เดือน


โมดูลที่ 3: คิววันนี้ (Today's Queue / Queue Management)
- UI แสดงผลแบบ Timeline หรือ Calendar View แบ่งตาม "ห้อง/เตียง" และ "หมอนวด"
- ตารางคิวลูกค้า: แสดงสถานะคิว (รอรับบริการ, กำลังนวด, เสร็จสิ้น, ยกเลิก)
- การจัดการคิว (CRUD): เพิ่มคิว Walk-in, แก้ไขรายละเอียด/เวลา, ยกเลิกคิว (ระบุเหตุผล)


โมดูลที่ 4: จองคิว (Booking Engine)
- Flow การจอง: เลือกบริการ -> เลือกวัน/เวลา -> เลือกหมอนวด (ระบุตัว/ไม่ระบุ) -> เลือกห้อง/เตียง
- Validation: ระบบต้องตรวจสอบการซ้อนทับกันของเวลา (Time Conflict) ของทั้งหมอนวดและห้อง/เตียง ก่อนบันทึก
- ดึงข้อมูลลูกค้าเก่าจากเบอร์โทรศัพท์ได้ทันที


โมดูลที่ 5: ใบเสร็จ / สลิป (Receipt & Invoice)
- สร้างและออกใบเสร็จรับเงิน/ใบกำกับภาษีอย่างย่อ
- พิมพ์สลิปผ่าน Thermal Printer (รองรับ ESC/POS)
- ฟังก์ชันส่ง e-Slip หรือรายละเอียดบิลให้ลูกค้าผ่าน LINE หรือ Email


โมดูลที่ 6: ลูกค้า (Customer CRM)
- จัดการฐานข้อมูลลูกค้า (CRUD): ชื่อ, เบอร์โทร, LINE ID
- Profile: บันทึกประวัติสุขภาพ ข้อควรระวัง (เช่น น้ำหนักมือที่ชอบ, จุดที่ห้ามนวด)
- History: แสดงประวัติการเข้าใช้บริการย้อนหลังทั้งหมด ผูกกับบิล POS


โมดูลที่ 7: สมาชิก (Membership)
- ระบบสมัครสมาชิกใหม่และผูกกับฐานข้อมูลลูกค้า
- Tier Management: กำหนดระดับสมาชิก (เช่น Silver, Gold, Platinum) พร้อมสิทธิพิเศษอัตโนมัติใน POS
- Reward Points: ระบบคำนวณแต้มสะสมจากยอดซื้อ (เช่น 100 บาท = 1 แต้ม) และระบบตัดแต้มเพื่อเป็นส่วนลด


โมดูลที่ 8: แพคเกจ (Packages)
- สร้าง Master Data แพ็กเกจ (เช่น นวดไทย 10 แถม 1, สปาคอร์ส 5 ครั้ง)
- ขายแพ็กเกจผ่านหน้า POS
- ระบบติดตามยอดคงเหลือ (Balance Tracking): เมื่อลูกค้ามาใช้บริการ ระบบจะตัดยอด (Redeem) อัตโนมัติและเก็บบันทึกประวัติการใช้ (ใครเป็นคนกดตัดยอด, วันเวลาใด)


โมดูลที่ 9: หมอนวด (Masseuse Management)
- จัดการข้อมูลส่วนตัวและรูปโปรไฟล์หมอนวด
- ตารางทำงาน (Shift Management): กำหนดวันเข้างาน, วันหยุด, เวลาพัก
- ดูสรุปรายได้และรอบคิวงานของแต่ละบุคคล


โมดูลที่ 10: คอมมิชชั่น (Commission System)
- ตั้งค่าสูตรคำนวณ: แบบเปอร์เซ็นต์ (% จากยอดบริการ) หรือ แบบ Fix Rate (บาท/ชั่วโมง)
- สามารถตั้งเงื่อนไขหักต้นทุนร้านก่อนคิดเปอร์เซ็นต์ได้
- ออกรายงานสรุปค่าคอมมิชชันรายวัน/รอบบิล เพื่อส่งให้บัญชีจ่ายเงิน


โมดูลที่ 11: สินค้า (Product Master Data)
- เพิ่ม/แก้ไข รายการสินค้า แยกเป็น 2 ประเภท: 1. สินค้าขายปลีกหน้าร้าน 2. สินค้า/วัตถุดิบใช้ภายใน (น้ำมันนวด, ลูกประคบ)
- กำหนด SKU, บาร์โค้ด, หมวดหมู่, ต้นทุน, ราคาขาย


โมดูลที่ 12: โปรโมชั่น (Promotion Engine)
- สร้างแคมเปญลดราคา กำหนดช่วงเวลา (Start-End Date)
- ระบบคูปอง (Promo Code): จำกัดจำนวนสิทธิ์การใช้, กำหนดยอดซื้อขั้นต่ำ


โมดูลที่ 13-116: ระบบรายงาน (Analytics & Reports)
- 13. ยอดขาย: สรุปรายวัน, รายเดือน, รายปี (ดูแบบกราฟและตาราง)
- 14. บริการ: สรุปบริการที่ขายดี, สัดส่วนรายได้ต่อประเภทบริการ
- 15. หมอนวด: สรุปรายได้และค่าคอมมิชชันของหมอนวดแต่ละคน, จำนวนรอบที่รับลูกค้า
- 16. สินค้า: สรุปสินค้าขายดี, รายงานมูลค่าสต็อกคงเหลือ
* ทุกรายงานต้องสามารถ Export เป็น .csv ได้


โมดูลที่ 17: การเงิน (Financial / Cash Flow)
- บันทึกรายรับ (Auto จาก POS)
- บันทึกรายจ่ายประจำวัน (Petty Cash/Manual Entry) เช่น ค่าแม่บ้าน, ค่าซื้อของเข้าร้าน
- รายงานสรุปกำไร-ขาดทุนเบื้องต้น (P&L)


โมดูลที่ 18: พนักงาน (Staff Management)
- จัดการข้อมูลพนักงานทั่วไป (ที่ไม่ใช่หมอนวด) เช่น แคชเชียร์, ผู้จัดการ
- กำหนดรหัสผ่านและสิทธิ์การเข้าใช้งานระบบ (RBAC)


โมดูลที่ 19: สาขา (Branch Management)
- เพิ่ม/แก้ไข ข้อมูลสาขา (ชื่อสาขา, ที่อยู่, เบอร์โทร)
- Data Isolation: ข้อมูลคิว สต็อก ยอดขาย จะต้องถูกผูกกับ Branch ID และไม่ปะปนกัน


โมดูลที่ 20: ผู้ใช้ระบบ (User Accounts)
- จัดการ Account สำหรับเข้าสู่ระบบทั้งหมด พร้อมระบบ Reset Password


4. ข้อกำหนดทางเทคนิค (Non-Functional Requirements)
- Database: Relational Database (MySQL) รองรับ Transaction ACID properties สำหรับ POS และ Inventory
- Architecture: Cloud-based รองรับการ Scale และ Multi-tenant 
- Frontend: PHP
- Backend: PHP/Laravel พัฒนาแบบ RESTful API
- Security: เข้ารหัสรหัสผ่าน (Bcrypt), ป้องกัน SQL Injection / XSS, และใช้งานผ่าน HTTPS (SSL/TLS)
